Binary Search Tree — O(log n)

"Double the data, take just one more step."

Nodes (n)
1
Depth (steps to find any item)
1
log₂(n) + 1
1
Each level doubles the number of nodes — but the depth only grows by one.
A tree with 1,000,000 items is only ~20 steps deep. That is what O(log n) looks like — the opposite of exponential.
