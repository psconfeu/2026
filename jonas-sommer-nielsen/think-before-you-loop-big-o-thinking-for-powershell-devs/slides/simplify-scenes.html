The three (and a half) rules

  • Drop constant coefficients — 3n is still O(n). Same shape, different slope.
  • Drop additive constants — n + 100 is O(n). Startup cost doesn't grow with the input.
  • Drop lower-order terms — n² + n is O(n²). The fastest-growing term eventually buries everything else.
  • Bonus: when one input is bounded, O(m + n) collapses to O(n). Only keep both if both can grow.

Why this works: Big O describes shape at scale. Slide the n slider above and watch the table — the ratio f(n)/g(n) approaches a constant in every case. Two algorithms with the same Big O scale the same way, even if one is 10x slower in absolute terms.
